# 설명 순서 원칙

기능을 설명할 때는 다음 순서를 따른다:
1. **구조** — 폴더/파일이 어디에 위치하는지
2. **파일 예시** — 실제 파일 내용이 어떻게 생겼는지
3. **생성 방법** — 어떻게 만드는지
4. **호출 방법** — 어떻게 사용하는지

# PPT/프레젠테이션 원칙

## 핵심 원칙

- **1 슬라이드 = 1 메시지** — 하나의 화면에는 하나의 정보만, 시각적으로 크게 전달한다
- **슬라이드는 빌보드처럼** — 청중이 3초 안에 핵심을 파악할 수 있어야 한다 (Nancy Duarte)
- **슬라이드 ≠ 문서** — 슬라이드에 모든 것을 담으려 하지 말 것. 발표자가 설명을 보충한다 (Garr Reynolds)

## 텍스트

- 폰트 크기 최소 30pt (Guy Kawasaki 10/20/30 Rule)
- 한 슬라이드에 글머리 기호 최대 3~4개, 한 줄에 5~6단어 이내
- 텍스트가 많으면 슬라이드를 나눈다 — 절대 욱여넣지 않는다

## 시각 디자인

- **여백을 충분히** — 빈 공간은 낭비가 아니라 메시지를 강조하는 도구 (Garr Reynolds)
- **시각 계층 구조** — 크기, 색상, 위치로 중요도를 표현한다
- **이미지는 슬라이드당 1개** — 메시지를 뒷받침하는 목적이 있어야 한다
- 불필요한 장식, 클립아트, 의미 없는 그래픽은 제거한다

## 구조와 분량

- 10/20/30 Rule (Guy Kawasaki): 10장 이내, 20분 이내, 30pt 이상
- 1분에 1슬라이드 — 20분 발표라면 약 20장
- 내용이 넘치면 쪼개고, 모자라면 비워둔다

## 전달력

- **STAR Moment** — 청중이 반드시 기억할 하나의 인상적 장면을 만든다 (Nancy Duarte)
- 애니메이션은 정보 전달을 돕는 도구로만 — 과하면 산만해진다
- 대비(Contrast)로 핵심을 부각한다 — 색상, 크기, 배치의 대비 활용

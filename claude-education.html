03 / 12 Concepts

왜 불필요한 정보가 성능을 떨어뜨리는가?

1
뭐가 중요한지 판단하기 어려워짐 정보가 많아질수록 AI가 핵심을 구분하지 못함
2
필요한 정보가 불필요한 정보에 묻혀서 놓침 중요한 규칙이 있어도 주변 노이즈에 가려짐
3
사람도 마찬가지 보고서 100페이지에서 핵심 3줄 찾기 vs 핵심만 정리된 1페이지
많이 넣는다고 좋은 게 아니라, 핵심만 넣어야 AI가 제대로 동작한다.

CLAUDE.md — "우리 팀 규칙서"

프로젝트 루트에 파일을 두면, 매 세션마다 자동으로 읽음

백엔드 Spring Boot

# 주문관리 API — Spring Boot 3.2
## 명령어
- ./gradlew test — 전체 테스트
- ./gradlew bootRun — 로컬 실행
## 컨벤션
- DTO: {동사}{도메인}Request
- @Transactional은 Service만
## 절대 하지 말 것
- System.out.println 금지
- prod.yml에 시크릿 하드코딩 금지

프론트엔드 React

# 어드민 대시보드 — React / TS
## 명령어
- npm run dev / lint / test
## 컨벤션
- 컴포넌트: PascalCase
- 폴더: feature 기반
  src/features/{기능}/
## 접근성
- img에 alt 필수
- 인터랙티브 요소 aria-label 필수
한번 적어두면, 매번 말 안 해도 알아서 따른다.

실제 프롬프트 예시

"orders 테이블에서 최근 7일간 일별 주문 건수 조회해줘"
"이번 스프린트 Jira 티켓 현황을 Confluence 주간보고에 정리해줘"
"사내 메신저 #dev-alerts 채널에서 오늘 에러 메시지 찾아줘"
"이 이슈에 대한 PR을 만들어줘"

주의: MCP는 대화 시작 시 활성화된 모든 MCP의 사용법을 컨텍스트에 포함시킴
→ 안 쓰는 MCP가 켜져 있으면 컨텍스트를 잡아먹는 주범 | /mcp 명령어로 확인 후 미사용 MCP 비활성화

Subagents — 전문가에게 위임

🔒

security-auditor

SQL 인젝션, XSS, 하드코딩 시크릿 탐지

🧪

qa-analyzer

영향 범위 분석, 리그레션 포인트 도출

Agent Teams — 동시 협업

결제 모듈을 리팩토링할 에이전트 팀을 만들어줘.
- API 담당: 결제 API 엔드포인트와 DTO 리팩토링
- DB 담당: 결제 테이블 마이그레이션 스크립트 작성
- 테스트 담당: 결제 관련 통합 테스트 작성
서로 발견한 내용을 공유하면서 진행해.
혼자 다 안 해도 된다. 팀을 꾸려서 시키면 된다.
